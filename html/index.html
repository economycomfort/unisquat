You got phished.

Why?

The hyperlink you clicked on contained a unicode character. Several unicode characters can look very similar, if not identical, to their ASCII cousins.
For instance, 'w' vs. 'ᴡ'. Can you tell the difference? Maybe with the right font (such as the default font on this web page.)
Could you tell the difference when this pops up in a tooltip in your email client, or Teams chat, or Discord community, as part of a domain name with an otherwise legitimate-looking URL?

Example:
Legitimate: https://walmart.com/shop/deals
Unisquatted: https://ᴡalmart.com/shop/deals

That's pretty tough.

How?

Domain names can be registered with Unicode characters in them, such as Cyrillic or even your favorite emojis.
In order to translate those characters into ASCII, a domain name is generated using Punycode, which is a method of representing Unicode characters within an ASCII domain name.
What you're actually registering, in this case, is an ASCII domain name that represents a Unicode character. In the case of my domain, ᴡalmart.com, it's actually represented in Punycode as xn--almart-245b.com.
Modern web browsers will now display the Punycode in the URL bar instead of the Unicode characters in an effort to tip users off.
However, modern operating systems still display the Unicode characters in a domain name in a tooltip, which most security-conscious users will look at to observe if the URL they're about to click is legitimate.
In this case, a user still has no realistic indication the domain they're about to go to isn't legitimate without performing additional investigation, which they'd have no real reason to do.
In reality, a user won't realize something is wrong until AFTER the click, and even then only if they're paying close attention to their browser.

What's the fix?

Unsure, but I'm not that smart. My guess is that operating system UX elements displaying a URL should display the Punycode instead of the Unicode, or provide some visual indicator that the domain name contains a Unicode characer.
In my opinion, this is still a problem.
